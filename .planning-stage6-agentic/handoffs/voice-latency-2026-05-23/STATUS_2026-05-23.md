The streaming gate is the AND of:
1. `source === 'confirmation'`
2. session has `streamConfirmations=true` (per-session snapshot)
3. iOS advertised `streaming_http_audio` capability (Stage 1b
   commit 1b.3 introduces this on iOS)
4. kill switch not active

**Until iOS Stage 1b ships, every session falls through to the legacy
batch path** because the capability gate is closed. This is deliberate:
the backend can deploy + the env flag can flip without breaking
old iOS builds.

## How to actually hit the 2–2.5s warm budget

Requires THREE things in concert:

1. **Backend Stage 2 deployed** ✅ done
2. **iOS Stage 1b + 2.1 + 2.2 shipped via TestFlight** ⏳ pending
3. **Two task-def env flips:**
   - `VOICE_LATENCY_STREAM_CONFIRMATIONS=true` (turns on the streaming path)
   - `VOICE_LATENCY_USE_MULTI_CONTEXT=true` (amortises the 800ms BOS handshake — the real warm win)

   Both must come via committed `ecs/task-def-backend.json` updates per
   CLAUDE.md MANDATORY infra-from-source rule. Cold path still gives
   ~3s; warm path (steady-state in a session) hits ~2.09s.
